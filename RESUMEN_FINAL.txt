╭─────────────────────────────────────────────────────────────────────────╮
│ 📐 SOLID PRINCIPLES APLICADOS                                           │
╰─────────────────────────────────────────────────────────────────────────╯

S️⃣ Single Responsibility
  • Cada servicio: una responsabilidad
  • BreakdownService → Firebase
  • CameraService → Cámara

O️⃣ Open/Closed
  • Abierto para extensión
  • Cerrado para modificación
  • Validadores extensibles

L️⃣ Liskov Substitution
  • Servicios intercambiables
  • Fácil de mockear
  • Compatible con tests

I️⃣ Interface Segregation
  • Interfaces específicas
  • Métodos agrupados lógicamente
  • Sin megainterfaes

D️⃣ Dependency Inversion
  • Inyección de dependencias
  • No instancias directas
  • Tokens de Angular
